(1) creating for trigger

Syntax :
CREATE (OR REPLACE] TRIGGER [User.] TriggerName
{BEFORE/AFTER/INSTEAD OF)
{DML Event on User. TableName/
DDL Event on [SCHEMA/DATABASEJ/
Database Event on [SCHEMA/DATABASE]}
[FOR EACH(ROW/STATEMENT} [WHEN (condition)]
DECLARE
Variable declarations;
Constant declarations;
BEGIN
PL/SQL Code;
EXCEPTION
Exception PL/SQL Block;
END;

The Terminology used in syntax is explained below :
OR REPLACE :

It recreates the trigger if it is already exists. This option can be used to change the definition of an existing trigger without first dropping it.

schema :

The schema that contains the trigger. If the schema is omitted, the Oracle engine creates the trigger in the users own schema.

triggername:

It is the name of the trigger to be created.

BEFORE :

It indicates that the Oracle engine fires the trigger before executing the triggering statement.

AFTER :

it indicates that the Oracle engine fires the trigger after executing the triggering statement.

INSTEAD OF:

It indicates that the Oracle engine executes the code written in trigger instead of performing the actions that invoked the trigger.


1. DML Event includes -

DELETE:

It indicates that the Oracle engine fires the trigger whenever a DELETE statement removes a row from the table.

INSERT :

It indicates that the Oracle engine fires the trigger whenever an INSERT statement adds a to table.

UPDATE :

It indicates that the Oracle er:gine fires the trigger whenever an UPDATE statement changes a value in any column of the table.

ON

It specifies the schema and name of the table, in which the trigger is to be created. If schema is omitted, the Oracle engine assumes the table is in the users own schema. A trigger cannot be created on a table in the schema SYS.

2. DDL Events Includes :

CREATE TABLE :

It indicates that the Oracle CREATE engine fires the trigger whenever CREATE TABLE TABLE command is issued.

ALTER TABLE :

It indicates that the Oracle engine fires the trigger whenever ALTER TABLE command is issued for altering a table.

DROP TABLE :

It indicates that the Oracle engine fires the trigger whenever DROP TABLE command is issued for removing a table.
