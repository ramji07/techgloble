(1) Domain Rule

Domain is a set of values for a field of table (or attribute) in a database. A domain constraint (also called domain) is a specification the values that can be given for a particular field (or attribute) in a table. The value that each field (or attribute) can be assigned is expressed as being one or more of the following form: a data type, a range of values or a value from a specified set.

Declaring an attribute to be, of a particular domain, acts as a constraint on the values that it can take. Domain constraints are tested by the system whenever a new data item is entered into the database. It possible for several attribute to have same domain. The proper defination of domain constraints not only allows us to test values inserted in the database, but also permits us to test queries, to ensure that the comparision made, makes sense.

(2) constraints

A name can be given to any constraint we specify. A system name is generated by oracle engine corresponding to the name given by the database administrator. The name generated by oracle engine is used for internal purpose and the name provided by database administrator is used for displaying purpose. If we do not provide any name unique, name by default is given by oracle engine.


(A) "Not Null" constraint : The NOT NULL column constraint ensures that a table column (or field) cannot be left empty.


Syntax :

Create table tablename ( fieldname1 datatype(size) NOT NULL, fieldname2 datatype(size), fieldname3 datatype (size) NOT NULL,......,);

Example : Create table Course (SubID Number(2), Subject Char(10), Faculty Char(25) NOT NULL);

The NOT NULL constraint can be given to more than one fields columns of table.



(B) "Unique" Constraint :The UNIQUE column constraint ensures that a table column (or field) does not contain any duplicate value.

Synatx :

Create tabel tablename ( fieldname1 datatype (size) UNIQUE, fieldname2 datatype (size) UNIQUE, fieldname3 datatype (size),..);

Example :

Create table Course (SubID Number(2), Subject Char(10) UNIQUE, Faculty Char(25) NOT NULL);

The UNIQUE constraint can be given to more than one fields or columns of a table.

The NOT NULL and UNIQUE constraints can be given to same column (or field) simultaneously.



(C) Primary Key Constraint : Primary key is a field or combination of fields chosen by the database designer for uniquely identifying any record in a table.

The Primary key constraint is a combination of both NOT NULL and UNIQUE constraint. In order to identify any record uniquely, in a table, all the values entered in the column should be unique and should not have any null values.

Syntax :

Create table tablename(fieldname1 datatype(size) Primary key, fieldname2 datatype(size), fieldname3 datatype(size),...);

Example :Create table course (SubID NUMBER(2) Primary key, Subject Char(10) UNIQUE, Faculty Char(25) NOT NULL);

Only one field can be defined as primary key in a table if we try to insert a duplicate value in a primary key filed, oracle gives an error, "ORA=00001: unique constraint (SCOOT.SYS_ConstraintName) violated.



(D) Referential Integrity or Foreign Key : foreign key is a column (or a group of columns) whose values are derived from the primary key or unique key of some other table.

The table in which the foreign key is defined is called a Foreign table or Detail table. The table that defined the primary or unique key and is referenced by the foreign key is called the Primary table or Master table.

The master table can be referenced in the foreign key defination by using the reference adverb. If the name of the column is not specified, by default oracle references the primary key in the master table.

Since foreign keys represent relationships between tables it is also known as Rreferential Integrity.



Rules for Referentail Intergrity/Foregin keyConstraint :

  • Foreign key must reference a PRIMARY KEY or UNIQUE column(s) in primary table.
  • Foreign key will automatically reference the PRIMARY KEY of the MASTER table if no column or group of columns is specified when creating the FOREIGN KEY.
  • Foreign key rejects an INSERT or UPDATE of a value, if a corresponding value does not currently exist in the master table.
  • Rejects a DELETE for the Master table corresponding records in the DETAIL table exsist.
