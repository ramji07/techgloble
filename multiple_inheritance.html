(1) Inheritance and Multiple Inheritance

(3) Inheritance :

An object oriented database schema usually requires a large number of classes. Often, however, several classes are similar.
For example, assume that we have an object oriented database for a bank apolication. We would expect the class of bank customers to be similar to the
class of bank employees, in that both define variables for name, address and so on. However, there are variables specific to employees (salary for example)
and variables specific to customers (credit-rating for example). It would be desirable to define a representation for the common variables in one place.
We can define one only if we combine employees and customers into one class.

To allow the direct representation of similarities among classes we need to place classes in a specialization hierarchy.
For instance, we say that employee is a specialization of person because the' set of all employees is a subset of the set
of all persons. That is, every employee is a person. Similarly, customer is a specialization of person.

Messages and methods are inherited in the same way that variables are inherited. An important benefit of inheritance
in object oriented systems is substitutability : Any method of a class (or a function that takes an object of a class
A as an argument) can equally well be invoked with any object belonging to any subclass B of A. This characteristic lead
to code reuse, since the say, A messages, methods and functions do not have to be written again for objects of class B.
For instance the message get-name() is defined for the class person and it can be invoked with a person object or with any object belonging to any subclass of person such as customer or officer.

class person {
string name;
string address;
string get-name;
string get-address;
int set-address (string new-address);
};
class customer isa person{
int credit-rating;
};
class employee isa person{
data start-data;
int salary;
int annual-salary();
int employement-length();
};
class officer isa employee{
int office-number;
int expense-account-number;
};

(I) Multiple Inheritance :

In most cases, a tree structured organization of classes is adequate to describe applications in tree structured organizations,
each class can have at most one superclass. However, there are situations that cannot be represented well in a tree
structured class hierarchy.

Multiple inheritance permits a class to inherit variables and methods from multiple superclass.
The class subclass relationships is represented by Directed Acyclic Graph (DAG) in which a class may have more
than one superclass.

For example, suppose employees can either temporary (for a limited period) or permanent. We may then create subclass
temporary and permanent of the class employee. The subclass temporary would have an attribute termination date specifying when the employement term ends.
The subclass permanent may have a method for computing contributions to a company pension plan, which is not applicable to temporary employees.
