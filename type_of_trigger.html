(1) Types of Triggers

A trigger's type is defined by the type of trigger action and by the level at which triggeris executed. While defining a trigger, the number of times the trigger action is to be executed can be specified. This can be once for every row affected by the triggering statement (such as might be fired by an UPDATE statement that updates many rows) or once for the triggering statement, no matter how many rows it affects.

(1) Row Level Triggers :

A row trigger is fired each time a row in the table is affected by the triggering statement. Row level triggers are most common type of trigger. They are generally used in data auditing applications.

For example, if an UPDATE statement updates multiple rows of a table, a row trigger is fired once for each row affected by the UPDATE statement. If the triggering statement affects no rows, the trigger is not executed at all. Row triggers should be used when some processing is required whenever a triggering statement affects a single row in a table.

(2) Statement Level Triggers :

A statement trigger is fired once for each transaction independent of the number of rows the triggering statement affects and also even if no rows are affected. Statement level triggers are the default type of trigger. Statement triggers should be used when a triggering statement affects rows in a table but the processing required is completely independent of the number of rows affected.

(3) Before and After Triggers :

Before and after keywords are used to specify trigger timing i.e. specifying when the trigger action is to be executed in relation to the triggering statement.

BEFORE amd ALTER apply to both row and the statement triggers.

BEFORE trigger execute the trigger action immediately before the triggering statement is executed.

ALTER trigger executes the trigger action after the triggering ststement is executed. ALTER TRIGGERS are used when we want the triggering ststement to complete before executing the trigger action.

Following triggers can be created usingabove given type and timing of triggers -

(i) Before Statement Trigger :

Before executing the triggering statement, the trigger action is executed.

(ii) Before Row Trigger :

Before modifying each row affected by the triggering statement and before appropriate integrity constraints, the trigger is executed if the trigger restriction either evaluated to TRUE or was not included.

(iii) After Statement Trigger :

After executing the triggering statement and applying any referred integrity constraints, the trigger action is executed.

(iv) After Row Trigger :

After modifying each row affected by the triggering statement and possibly applying appropriate integrity constraints, the trigger action is executed for the current row if the trigger restriction either evaluates to TRUE or was not included. Unlike BEFORE row triggers, AFTER row triggers have rows locked.

(4) Instead of Triggers :

INSTEAD of Triggers are used to tell the Oracle engine what to do instead of performing the actions that invoked the trigger. The code block of INSTEAD of Trigger is executed in place of insert, update or delete commands issued by INSTEAD of trigger can be used on either object views or on relational views.

(5) Schema Triggers :

Schema triggers can be used to create triggers on schema operations. The triggering events for schema triggers are table, alter table and drop table. Schema triggers are used to prevent DDL operations and to supervise by providing additional security by monitoring the occurence of DDL operations. By schema triggers you can prevent users from creating & dropping their ocon table. These types of triggers are allowed in Oracle 8i and higher versions.

(6) Database Level Triggers :

Database level triggers can be used to create triggers to be fired on database events, including errors, logons, logoffs, startups and shutdown. This type of trigger can be used to automate database maintenance.
